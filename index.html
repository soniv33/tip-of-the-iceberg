🧊 tip-of-the-iceberg

Watch Apache Iceberg's core trick — it decides which files it can skip without opening them, using nothing but each file's min & max.

Explain it like I'm five
  1. Every time we save movies, they go in a new box (file), and we write on the lid the first and last title inside (min & max).
  2. To find "Margin Call", we read the lids first. If a box says Amelie–Drive, "Margin Call" can't be inside — skip it, don't open it.
  3. LIKE 'Ma%' (starts-with) still works from the lids. But LIKE '%ll' (ends-with) can't — we must open every box.
  4. Adding a movie makes a brand-new box; we never rewrite old ones.
  5. Because old boxes never change, we can time-travel to how the shelf looked before.
